# 文学作品原文查找策略

当用户询问特定文学段落（如"《少年》第一部第二章第三节"）时，按以下顺序查找原文。

## 查找层级

| 优先级 | 来源 | 方法 |
|--------|------|------|
| 1 | **Obsidian 笔记** | `search_files` 搜索 vault 中的读书笔记、书摘、读书会纪要 |
| 2 | **会话历史** | `session_search` 查找之前讨论过该作品的对话 |
| 3 | **中文在线全文** | 搜索 diancang.xyz、QQ阅读等免费源 |
| 4 | **英文 Gutenberg** | `curl gutenberg.org/cache/epub/<id>/` 获取英译本 |
| 5 | **凭记忆分析** | 诚实告知用户无法获取原文，提供记忆中的内容分析 |

## 中文在线源实测

| 源 | 状态 | 说明 |
|----|------|------|
| diancang.xyz | ⚠️ 不稳定 | 有部分章节，但导航混乱，搜索索引不全 |
| 99csw.com / 99lib.net | ❌ Cloudflare 拦截 | headless browser 无法通过 |
| QQ阅读 (book.qq.com) | ❌ 付费墙 | 章节目录可见，正文需会员 |
| ilibrary.ru (俄文) | ❌ 超时 | 可能被俄罗斯 IP 封锁 |
| 360 搜索 | ✅ 可用 | 用于发现中文源，再用 `browser_navigate` 访问 |

## 关键陷阱

- **Gutenberg 编号要验证**：`pg37536` 是《死屋手记》而非《少年》。《少年》英译本标题为 "A Raw Youth" 或 "The Adolescent"。
- **章节结构因译本不同**：中文版通常分"部-章-节"（第一部第二章第三节），英译本分 "Part-Chapter-Section"，结构对应但编号可能偏移。
- **QQ阅读章节目录是免费可查看的**：即使正文被付费墙挡住，章节目录列表可以作为结构验证。
- **⚠️ 部数混淆致死**：用户说"第一部第二章第三节"，拿到"第二部第三章"就以为是同一章——**先确认部（Part）编号再展开分析。** 宁可多问一句"是第一部的第二章对吗？"也不要在错误章节上展开长篇。
- **中文在线源的章节索引不全**：diancang.xyz 可能有第二部但没有第一部，360 搜索未必能索引到所有子章节。遇到缺章应快速切换策略，不要反复尝试同一源的不同 URL 变体。

## 微信读书集成

微信读书 skill (`weread`) 支持搜索书籍、查看笔记/划线/书评，但**不支持读取正文全文**。需要 `WEREAD_API_KEY` 环境变量（格式 `wrk-xxxxxxxx`）。若用户有 API Key 但未设置，提示 `export WEREAD_API_KEY=<key>`。

## 回退策略

当所有在线源不可用时：
1. 诚实告知限制
2. 提供基于记忆的内容分析（标注"凭记忆"）
3. 建议用户拍照/截图发来，用 OCR 进行逐段分析
4. 提供 `clarify` 让用户选择方案
